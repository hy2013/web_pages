# 移动端竖屏Web小游戏生成器

## 核心要求

**生成的游戏必须满足以下条件：**
- **手机端**：适配手机浏览器，界面和交互针对手机优化
- **触控操作**：完全通过触控进行游戏操作，不依赖鼠标或键盘
- **竖屏显示**：专为竖屏手机设计

## 功能介绍

该技能可以根据用户提供的游戏类型、核心玩法和视觉风格偏好，自动生成符合移动端竖屏显示规范的完整Web游戏页面代码。

## 使用方法

### 输入格式

用户需要提供以下信息：
1. **游戏类型**：如跑酷、射击、益智、休闲、消除、贪吃蛇、俄罗斯方块等任意类型
2. **核心玩法**：简要描述游戏的主要玩法机制
3. **视觉风格**：如像素风、卡通风格、简约风格、水墨风格、赛博朋克等
4. **其他特殊要求**（可选）：如颜色偏好、游戏难度等

### 输出结果

技能将生成一个包含时间戳的独立目录，目录中包含完整的HTML游戏文件：
- 目录名称格式：`game-<游戏类型>-<时间戳>`
- 完整的HTML文件
- 响应式竖屏布局，适配手机屏幕
- 纯触控操作支持

### 集成到主页面

生成游戏后，需要手动将游戏路径添加到主index.html的文件列表中：

在 `index.html` 的 `getAllFiles()` 函数中添加：
```javascript
{
    name: "游戏名称",
    path: "game-xxx/index.html"
}
```

## 技术规范

### 触控操作
- 屏幕任意位置点击/滑动均可触发游戏操作
- 支持单指点击、长按、滑动等触控手势
- 触控区域覆盖整个游戏界面
- 无需任何按钮，简化操作

### 竖屏适配
- 强制竖屏布局，宽度适配手机屏幕（通常320px-428px）
- 自适应全屏，使用100vw × 100vh
- 屏幕比例采用 16:9 或 长竖屏比例

### 响应式设计
- 适配主流移动设备屏幕尺寸
- 使用相对单位确保布局适应不同屏幕
- 优化触控区域大小（至少44px × 44px）

## 注意事项

1. 生成的游戏仅支持触控操作，不考虑鼠标或键盘
2. 必须在竖屏模式下测试
3. 建议在手机浏览器中进行最终测试
